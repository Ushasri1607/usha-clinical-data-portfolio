# Healthcare Database Design (EHR Schema)

## Overview
This project focuses on designing a normalized relational database for a healthcare Electronic Health Record (EHR) system.  
The schema supports patient demographics, encounters, diagnoses, providers, and billing information.

## Objectives
- Design a normalized healthcare database
- Define primary and foreign key relationships
- Support clinical and operational reporting
- Follow healthcare data best practices

## Tools
- SQL (SQLite / SQL Server compatible)
- DB Browser for SQLite
- GitHub

## Key Entities
- Patients
- Encounters
- Providers
- Diagnoses
- Billing

## Deliverables
- Database schema (ER-style design)
- SQL DDL scripts
- Sample queries
